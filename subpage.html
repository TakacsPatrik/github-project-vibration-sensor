Overview - Problem Statement

The WITMOTION WT901C485 is a high-precision vibration sensor that collects data along 9 axes: it measures acceleration, angular velocity, and the magnetic field, and from these values it calculates tilt angles and heading. In addition, it measures temperature, records the chip’s internal timestamp, and during program execution it also associates the data with the current system time. The device communicates via the RS-485 (Modbus RTU) interface, which allows reliable integration into industrial or research systems and supports the simultaneous networking of multiple sensors.

In this project, I mounted the WT901C485 vibration sensor on a CNC lathe in order to collect real-time data at a sampling frequency of 5 Hz (corresponding to a sampling interval of 0.2 seconds) about the machine’s operation. A threshold value was assigned to the vibration data, and whenever this threshold was exceeded, the program triggered an external warning light to flash. For data evaluation, I also developed a graphical user interface (GUI), which allows the measured values to be easily monitored and tracked over time.

Development process of the data gathering

During the project, I first connected to the device via serial communication and configured its parameters. After that, I calculated a resultant value from the vibration data measured along the three axes, which I then collected together with the other values in a list. The gathered data is saved every 10,000 rows into a .csv file, each file tagged with a timestamp. If the machine’s vibration exceeds the set threshold, the program uses HID communication to connect to an external flasher, sending it a high value to activate it. Otherwise, every five seconds the flasher receives a low value.

Development process of the GUI

The goal of the project was to create a simple interface that displays the measurement results on a graph within a user-defined time interval. In addition to specifying the start and end times, the user can also set a threshold value and choose whether to apply data smoothing. Smoothing is enabled by default and uses a 5-sample moving average (which corresponds to about 1 second of smoothing at a 5 Hz sampling rate). On the graph, the current threshold is shown as a green dashed line, while periods exceeding the threshold are highlighted with red 3-minute bands. In the case of large gaps between samples, the system inserts interruptions (NaN values) so that these breaks are also visible on the plot. There is also an option to export the graph.
